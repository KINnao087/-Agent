## 1. 产品概要介绍

科技合同审核系统是一个基于 AI 大语言模型与计算机视觉技术的智能合同审核平台。系统能够自动对科技合同（技术开发、技术服务、技术咨询等）进行全方位审查，主要特点有：审核流程自动化、识别准确率高、支持多种合同格式、审核结果可追溯、支持交互式对话与 API 两种使用方式。

本系统的主要功能包括：合同基础信息提取与平台数据核对、合同文本完整性检测（连续性、完整性、替换页、清晰度）、甲乙双方签章检测、骑缝章检测、合同主体真实性/有效性审查，以及自动生成结构化审核报告（JSON + Markdown）。系统支持 PDF、PNG、JPG、BMP、WebP 等多种文件格式输入。

## 2. 快速入门

### 2.1 环境配置

操作员首次使用系统前，需要完成以下配置步骤：

（1）安装 Python 3.11 或更高版本；

（2）创建虚拟环境并安装项目依赖：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[dev]"
```

（3）安装 PaddleOCR 后端（CPU 版本）：

```powershell
python -m pip install paddlepaddle
```

如有 NVIDIA GPU，可安装 GPU 版本以获得更快的 OCR 处理速度：

```powershell
python -m pip install paddlepaddle-gpu
```

（4）在项目根目录创建 `.env` 文件，配置 AI 模型密钥（详见 3.1 节）。

### 2.2 启动交互式 CLI

操作员配置完成后，在项目根目录执行以下命令启动 AI 智能对话终端：

```powershell
python main.py
```

启动后将进入基于 Textual 的 TUI（终端用户界面）Shell。操作员可以用自然语言与 AI 对话，AI 会自动调用审核工具完成各项审核任务。

### 2.3 合同基础信息核对

操作员在 CLI 中输入合同文件路径，AI 会自动调用 `check_basic_info` 工具，从合同中提取 20 个关键字段（合同编号、项目名称、签订日期、合同周期、成交金额、技术交易金额、支付方式、甲乙双方名称/法人/电话/地址/经办人等），并与平台录入数据进行逐字段比对，输出匹配/不匹配的比对结果。

### 2.4 文本完整性检测

AI 调用 `check_text_integrity` 工具，对合同文本进行四项检查：连续性检查（识别文本断页）、完整性检查（必要条款是否缺失）、替换页检测（是否存在被替换的页面）、清晰度检查（文字是否模糊或遮挡）。检查结果会汇总到最终审核报告中。

### 2.5 签章检测

AI 调用 `check_contract_seals` 工具，利用混合视觉检测算法（OpenCV 特征提取 + 逻辑回归分类器）对合同页面进行图像分析，自动识别并定位甲乙方公章、签字等签章，判断印章数量、清晰度和完整性。

### 2.6 骑缝章检测

AI 调用 `check_cross_page_seal` 工具，检测合同跨页骑缝章的存在性与完整性，验证骑缝章是否跨越多页形成完整图案。

### 2.7 合同真实性审查

AI 调用 `check_contract_authenticity` 工具，通过企查查查询合同主体的工商注册信息，并通过 Tavily 网络搜索核实主体公开信息，检查主体是否存在、经营状态是否正常，识别经营异常、失信记录等有效性风险。

### 2.8 审核报告生成

当所有审核步骤完成后，AI 调用 `write_review_report` 工具，汇总各步骤的审查结果，生成两份报告：JSON 格式（`contract_review.json`）适合程序二次处理，Markdown 格式（`contract_review.md`）适合打印签字、邮件发送或存档查阅。报告保存在 `artifacts/reviews/<review_id>/reports/` 目录下。

## 3. 详细的使用说明

### 3.1 环境变量配置

操作员需要在项目根目录创建 `.env` 文件，填入以下配置：

```
# ===== 主脑 AI 配置（必填）=====
MAIN_AI_API_KEY=sk-your-deepseek-api-key
MAIN_AI_BASE_URL=https://api.deepseek.com
MAIN_AI_MODEL=deepseek-v4-flash

# ===== 文本审核 AI 配置（必填）=====
TEXT_AI_API_KEY=sk-your-deepseek-api-key
TEXT_AI_BASE_URL=https://api.deepseek.com
TEXT_AI_MODEL=deepseek-v4-pro

# ===== 视觉审核 AI 配置（可选）=====
VISION_AI_API_KEY=
VISION_AI_BASE_URL=
VISION_AI_MODEL=

# ===== 网络搜索（必填）=====
TAVILY_API_KEY=tvly-dev-your-tavily-key

# ===== 企查查 Cookie（可选）=====
QCC_COOKIE=your-qcc-cookie

# ===== LangSmith 追踪（可选，调试用）=====
LANGSMITH_TRACING=true
LANGSMITH_ENDPOINT=https://api.smith.langchain.com
LANGSMITH_API_KEY=lsv2_pt_your-langsmith-key
LANGSMITH_PROJECT=contract_agent
```

各配置项说明如下：

| 变量组 | 用途 | 是否必填 |
|--------|------|----------|
| `MAIN_AI_*` | 主脑模型，负责对话理解、工具调用决策和报告组装 | 必填 |
| `TEXT_AI_*` | 文本审核模型，负责基础信息提取和文本完整性审查 | 必填 |
| `VISION_AI_*` | 视觉审核模型，负责图片多模态审核 | 可选，留空时回退到 `config/config.json` |
| `TAVILY_API_KEY` | Tavily 网络搜索 API 密钥 | 必填 |
| `QCC_COOKIE` | 企查查登录 Cookie，用于查询企业工商信息 | 可选（推荐填写） |

AI 模型支持任何兼容 OpenAI 接口的服务商。操作员可将 `*_AI_BASE_URL` 指向 DeepSeek、DashScope 或其他兼容服务。

### 3.3 完整合同审核流程

操作员输入合同审核指令，AI 会自动编排并执行全部审核步骤：

```
👤 用户: 请审核这份合同：D:\contracts\2024-技术开发合同.pdf，
        平台数据：甲方"北京科技有限公司"，乙方"上海信息技术有限公司"，
        成交金额150万元，签订日期2024-03-15

🤖 AI: 好的，我来为您审核这份合同。首先准备合同材料...

        [1/6] prepare_contract      —— 准备合同材料，执行OCR识别...
        [2/6] check_basic_info       —— 提取基础信息，与平台数据比对...
        [3/6] check_text_integrity   —— 检查文本连续性和完整性...
        [4/6] check_contract_seals   —— 检测甲乙双方签章...
        [5/6] check_cross_page_seal  —— 检测骑缝章...
        [6/6] write_review_report    —— 生成审核报告...

        ✓ 审核完成！报告已保存至：
          artifacts/reviews/abc123/reports/contract_review.json
          artifacts/reviews/abc123/reports/contract_review.md
```

AI 按以下顺序自动完成审核流程：

```
prepare_contract  →  check_basic_info
                  →  check_text_integrity
                  →  check_contract_seals
                  →  check_cross_page_seal
                  →  check_contract_authenticity
                  →  write_review_report
```

操作员无需手动指定每一步——只需提供合同文件和平台数据，AI 自动判断需要执行哪些步骤。

### 3.8 结构化解析 CLI

除了交互式对话，系统还提供命令行工具直接对文档进行 OCR 和结构化解析，不执行审核流程：

```powershell
python main.py parse --file <PDF或图片目录路径>
```

**示例**：

```powershell
# 解析单个 PDF
python main.py parse --file "D:\contracts\技术合同.pdf"

# 解析图片目录
python main.py parse --file "D:\contracts\合同扫描件\"
```

系统将自动执行以下处理流程：
1. PDF → 按页渲染为 PNG 图片
2. 图片 → PaddleOCR 识别文字
3. AI → 整理为结构化 JSON 输出

**支持的输入格式**：

| 格式 | 处理方式 |
|------|----------|
| PDF | 按页渲染为 PNG |
| PNG | 直接使用 |
| JPG / JPEG | 转换为 PNG |
| BMP | 转换为 PNG |
| WebP | 转换为 PNG |
| 目录 | 按文件名顺序展开其中的支持文件 |

所有转换结果会缓存到输入路径旁的 `_normalized_images/` 目录中。

### 3.9 线性化 CLI

将合同文档转换为线性文本表示，便于后续处理或人工审阅：

```powershell
python main.py linearizer --file <PDF或图片目录路径> --output-dir <输出目录>
```

**参数说明**：

| 参数 | 说明 | 必填 |
|------|------|------|
| `--file` | 合同 PDF 文件路径或包含图片的目录路径 | 是 |
| `--output-dir` | 输出目录路径 | 是 |

### 3.10 HTTP API 服务

系统提供 HTTP REST API，支持程序化调用审核功能。

**启动服务**：

```powershell
python -m uvicorn server:app --reload
```

服务启动后，访问 `http://127.0.0.1:8000/docs` 打开 Swagger UI 交互式文档。

**自定义端口**：

```powershell
python -m uvicorn server:app --host 0.0.0.0 --port 8080 --reload
```

**API 接口**：

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/contracts/check-basic-info` | 合同基础信息核对 |

**基础信息核对接口示例**：

请求：

```json
{
  "contract_text": "合同编号：JK-2024-001\n项目名称：智能安防系统开发\n...",
  "platform_basic_info": {
    "contract_no": "JK-2024-001",
    "project_name": "智能安防系统开发",
    "sign_date": "2024-03-15",
    "deal_amount": 1500000.00,
    "seller_name": "北京科技有限公司",
    "buyer_name": "上海信息技术有限公司"
  }
}
```

响应：

```json
{
  "contract_basic_info": {
    "contract_no": "JK-2024-001",
    "project_name": "智能安防系统开发",
    "sign_date": "2024年3月15日",
    "deal_amount": 1500000.00,
    ...
  },
  "compare_result": {
    "contract_no": { "match": true, "contract_value": "JK-2024-001", "platform_value": "JK-2024-001" },
    "deal_amount": { "match": true, "contract_value": "150万元", "platform_value": "1500000.00" },
    ...
  },
  "summary": "共20个字段，匹配18个，不匹配1个，平台未提供1个。"
}
```
